# Fuyutsui（“把复杂交给像素，把快乐留给手”）

这是一个给《魔兽世界》的“小助手组合包”：一边在游戏内用 Lua 把关键信息编码进屏幕上的**一条彩色像素条**，另一边在电脑桌面用 Python 读像素、按职业/专精匹配策略，然后通过热键把“该施放的技能”送到你手上（或者说：送到你键位上）。

它的哲学很简单：  
你负责玩，插件负责把复杂度压缩成“可读的状态”；至于像素在干嘛——像素也很忙，它们只负责当“翻译官”。

---

## 它到底做了什么？

- **Lua（游戏内）**：根据你的职业/专精、当前战斗/移动/施法状态、目标与队伍信息、技能冷却等，更新屏幕顶部的一条“色块/色条”数据。
- **Python（桌面端）**：扫描这条色条，依据 `config.yml` 把每个像素值还原成一个结构化的 `state_dict`（包含队伍成员、光环、技能冷却等）。
- **策略决策（职业逻辑）**：根据当前职业专精加载对应的逻辑模块（例如圣骑/牧师等），从 `state_dict` 里判断“下一步该干嘛”。
- **热键触发**：用 `keymap/*.yml` 把 `(单位, 技能)` 映射到一个热键字符串，再用 Windows 的 `PostMessage` 把热键发给标题为“魔兽世界”的窗口。

结果就是：你不用盯着一堆 UI 指示器读空气，它帮你把“读空气”变成了“读像素”，再把“该点什么”变成“该按哪个键”。

---

## 工作原理（像素通信协议 · Lua <-> Python）

1. **Lua 端创建色条**
   - `core/block.lua` 会创建一个位于屏幕顶端的 `colorBars`（255 个块，高度 1 像素），并通过 `fu.updateOrCreatTextureByIndex(i, b)` 更新每个块的颜色。
   - Lua 通过颜色通道编码数据：每个块用 **G 通道作为索引（i/255）**，用 **B 通道作为数值（b）**。

2. **Python 端扫描色条**
   - `GetPixels.py` 会先找到起始像素 `(R=0, G=1, B=0)`，然后从起点向右扫描像素。
   - 以 **G 通道作为索引（1~255）**、以 **B 通道作为数值**，还原出一行 `row_data`。

3. **按配置还原结构化状态**
   - `config.yml` 定义 `state:` 的字段“每个字段在哪个 step 上”，以及每个职业专精的 `spells` 与 `group` 结构。
   - `build_state_dict()` 把 `row_data[step]` 解析成：
     - 顶层字段（例如“生命值/能量值/战斗/移动/施法/引导”等）
     - `spells`（技能冷却/可用性等）
     - `group`（队伍成员：每个成员对应一组字段）

---

## 配置文件怎么用？

- `config.yml`
  - 定义通用 `state`（比如职业/专精/有效性等）的映射 step。
  - 对每个 `class_id -> spec_id` 给出该专精关心的：
    - `spells`：技能名 -> step 映射
    - `group`：队伍成员字段 -> step 映射（队伍人数、字段列表等）

- `keymap/*.yml`
  - 定义 `(unit, spell)` -> 热键字符串。
  - 例如在 `keymap/paladin.yml` 里，你会看到类似：`unit: 1, 技能: 圣光闪现, 热键: CTRL-''` 这样的映射。

当你觉得“我今天就是要用某个技能去某个队友身上”的时候，就该改这里，而不是改脑子。

---

## 职业逻辑模块：谁决定“下一步按什么”？

Python 会根据职业 ID 加载对应的逻辑函数（在 `Fuyutsui/class/*_logic.py`）。

逻辑模块通常会做这些事情：

- 从 `state_dict` 里读冷却/目标有效性/队伍生命值/光环情况
- 按规则挑选一个或多个“目标单位”
- 如果命中条件，就调用 `get_hotkey(unit, spell)` 找到热键并返回：
  - `action_hotkey`（要按的键）
  - `current_step`（面板上显示的“我准备干嘛”）

## 如何启动（建议从“别急”开始）

1. **确保游戏内启用 AddOn**
   - 把 `Fuyutsui` 放到 WoW 的 `Interface/AddOns/` 目录，并在游戏内启用。
   - 启用后，Lua 会开始在屏幕顶部绘制色条数据。

2. **安装 Python 依赖**
   - 执行：`pip install -r requirements.txt`
   - 注意：`GetPixels.py` 里使用了 `mss`，而当前 `requirements.txt` 只写了 `customtkinter` 和 `PyYAML`，所以你大概率还需要额外装一下：
     - `pip install mss`

3. **运行桌面 GUI**
   - 运行：`python logic_gui.py`
   - 页面右上角有 `逻辑开启` 勾选框。

4. **开/关逻辑**
   - 你可以用 GUI 勾选框控制，也可以通过鼠标侧键 `XBUTTON2` 进行切换（程序里默认用它当“物理开关”）。

## 常见问题（让它不要“读错像素”）

- **窗口标题必须匹配**：热键发送默认找标题为“魔兽世界”的窗口。
- **分辨率/缩放会影响扫描**：Python 用 `GetClientRect` + DPI 适配来定位色条位置，但极端分辨率/缩放仍可能导致扫描偏移。
- **PostMessage 不一定对所有输入方式生效**：部分游戏/输入链路可能不响应，需要你再做微调。
